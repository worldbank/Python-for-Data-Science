    <article class="markdown-body entry-content p-5" itemprop="text"><h1><a id="user-content-writing-clean-code-based-on-pep8-guidelines" class="anchor" aria-hidden="true" href="#writing-clean-code-based-on-pep8-guidelines"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Writing clean code based on PEP8 guidelines</h1>
<p>Clear and well-written code is important. The main set of conventions for Python, written by Guido Van Rossum, is called <a href="https://www.python.org/dev/peps/pep-0008/#prescriptive-naming-conventions" rel="nofollow">PEP8</a>.</p>
<p>For our course, remember the following:</p>
<p><strong>1. Maximum line length</strong></p>
<ul>
<li>Lines of code shouldn't exceed 79 characters.</li>
<li>To comply with this, break code into several lines by wrapping expressions in parentheses.</li>
</ul>
<p><strong>2. Imports</strong></p>
<ul>
<li>Import libraries at the top of your notebook. Use a new line for each import.</li>
<li>Learn the conventional shortened names for common libraries (eg. import pandas as pd).</li>
</ul>
<pre>YES:
import sys
import os
</pre>
<pre>NO: 
import sys, os
</pre>
<p><strong>3. Declaring strings</strong></p>
<ul>
<li>Python lets you use single quotes or double quotes around strings; pick one option and stick to it.</li>
</ul>
<pre>YES:
string1 = 'I love Python'
string2 = 'not R'
</pre>
<pre>NO:
string1 = "I love Python"
string2 = 'not R'
</pre>
<p><strong>4. Whitespace</strong></p>
<ul>
<li>Use spaces between operators</li>
<li>Be consistent and don't add redundant whitespace.</li>
</ul>
<pre>YES:
a = b + c
spam(ham[1], {eggs: 2})
</pre>
<pre>NO:
a=b+c
spam( ham[ 1 ], { eggs: 2 } )
</pre>
<p><strong>5. Comments</strong></p>
<ul>
<li>Start comments with # and a single space.</li>
</ul>
<pre>YES:
x = x + 1          # Increment x
</pre>
<p><strong>6. Naming</strong></p>
<ul>
<li>Function and variable names should be lowercase, with underscores between the words as necessary to improve readability.</li>
<li>Use descriptive variable names.</li>
</ul>
<pre>YES:
mean_trip_time = []
</pre>
<pre>PROBABLY NOT:
mtt = []
</pre>
